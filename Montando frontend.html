Etapa 4 — Frontend Vue

Nesta etapa, vamos configurar o frontend com Vue 3 para consumir os dados do backend PHP, utilizando o endpoint /backend/index.php/tasks.

1️⃣ Estrutura das pastas do frontend

frontend/
│
├── src/
│   ├── components/
│   │   ├── TaskList.vue
│   │   ├── TaskForm.vue
│   │   └── TaskItem.vue
│   ├── api/
│   │   └── taskApi.js
│   ├── views/
│   │   └── Home.vue
│   └── App.vue
│
├── package.json
└── vite.config.js

2️⃣ Criação do projeto Vue

Execute o comando abaixo para criar o projeto:

npm create vue@latest frontend

Escolha as opções padrão (sem TypeScript, sem Router, sem Pinia, sem Testes).

Depois entre na pasta:

cd frontend

Instale as dependências:

npm install

E rode o servidor local:

npm run dev

O projeto deverá abrir em algo como: http://localhost:5173

Obs. Quando você fechar o terminal, o servidor do Vue vai parar.

Da próxima vez que quiser rodar o projeto, basta entrar na pasta e iniciar manualmente:

    cd frontend
    npm run dev

3️⃣ Conexão com o Backend

Vamos criar um arquivo taskApi.js que fará as requisições HTTP usando fetch().

// src/api/taskApi.js
const API_URL = "http://localhost/backend/index.php/tasks";

export async function getTasks() {
  const response = await fetch(API_URL);
  return await response.json();
}

export async function addTask(title) {
  const response = await fetch(API_URL, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title })
  });
  return await response.json();
}

export async function deleteTask(id) {
  const response = await fetch(`${API_URL}?id=${id}`, { method: "DELETE" });
  return await response.json();
}

⚙️ O Vue agora consegue consumir o backend via endpoint /backend/index.php/tasks.
